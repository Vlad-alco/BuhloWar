## Задача
Сделать две компоновки SVG-аппарата, переключаемые по режиму процесса:
- **Ректификация** (process=2) — текущий вид: дефлегматор сверху, царга с тарелками, бак снизу
- **Дистилляция** (process=1) — перевёрнутый дефлегматор слева, П-образная труба из нержавейки 2" соединяет верх бака с верхом дефлегматора, бутыль под дефлегматором
- **IDLE** (process=0) — показывать последний активный режим (запоминать в localStorage)
- **Главное**: анимация потока воды в дефлегматоре (`.coilflow`) должна работать только когда `waterValveOpen === true` — в обоих режимах
- Заменить примитивный бак на реалистичный бак Domspirt 37L (по образцу domspirt.ru): домообразная крышка с три-клампами, 2 боковые ручки с чёрными грипами, цифровой термометр, сливной кран с синей ручкой, клапан безопасности

## Решение

### Что было сделано

1. **Структура SVG** — разделена на две группы внутри одного `<svg>`:
   - `<g id="apparatusRect">` — ректификация (видна по умолчанию)
   - `<g id="apparatusDist" style="display:none">` — дистилляция (скрыта по умолчанию)
   - Общие `<defs>` с градиентами для стекла, меди, стали и нового бака

2. **Новый бак Domspirt 37L** (в обеих группах, через `transform="translate(cx, cy)"`):
   - Корпус: вертикальный цилиндр из браш-стали (`url(#tankBody)` — 7-стоповый градиент с вертикальными бликами)
   - Домообразная крышка (`url(#tankLid)`) с центральным 2" три-кламп феррулом
   - 2 три-клампа по бокам от феррула (с красными болтами `#c93030`)
   - Красный клапан безопасности слева на куполе
   - 2 боковые U-образные ручки (`url(#steelV)`) с чёрными резиновыми грипами (`#1a1a1a`)
   - Цифровой термометр слева на корпусе (белый корпус `#e8e8e8`, синий экран `#3a5a8a`, зелёные цифры `#7ff0cf`)
   - Сливной шаровый кран с синей пластиковой ручкой (`#2a6fc9`) справа внизу
   - Стальные кольца-усиления сверху и снизу корпуса
   - Трапециевидное основание (`url(#tankBase)`)

3. **Перевёрнутый дефлегматор в dist-режиме**:
   - Использован `transform="translate(-144, 364) rotate(180 200 98)"` для поворота на 180° вокруг центра (200, 98) и сдвига влево-вниз
   - После transform: патрубки воды снизу (y=510), вход пара сверху (y=410)
   - Змеевик и анимация `.coilflow` сохраняются (работают в обоих режимах через `document.querySelectorAll('.coilflow')`)

4. **П-образная труба из нержавейки 2"**:
   - Внешний контур: `stroke="url(#pipeSteel)"` шириной 18px с радиусами 14px на изгибах
   - Внутренний блик: `stroke="#e8eef2"` шириной 3px, opacity 0.55
   - Тёмная тень: `stroke="#1a2028"` шириной 2px, смещение (3, 3)
   - Феррул 2" три-кламп на стыке с баком
   - Путь: `M248 502 L248 394 Q248 380 234 380 L70 380 Q56 380 56 394 L56 410`
   - Также определён как `<path id="pipePath">` в `<defs>` для использования в `<animateMotion>`

5. **Анимация пара по П-трубе**:
   - 8 частиц (круги с `url(#vaporg)`)
   - Каждая использует `<animateMotion>` с `path=pipePath` для движения по пути бак→дефлегматор
   - Дополнительная пульсация opacity через `<animate>` (0→0.9→0.9→0)
   - Случайные длительности (2.4-3.6s) и задержки для естественного движения

6. **Логика переключения режимов в `updateApparatusSVG()`**:
   - Определение `displayProcess`: если `currentData.process` = 1 или 2 — его; иначе `lastActiveProcess` из localStorage
   - Запоминание: `localStorage.setItem('lastActiveProcess', ...)` при активном процессе
   - По умолчанию (если ничего не запомнено) — ректификация (process=2)
   - Переключение `display` стиля групп `apparatusRect` и `apparatusDist`

7. **Анимация потока воды (главное требование!)**:
   - `document.querySelectorAll('.coilflow').forEach(c => { c.style.animationPlayState = waterOn ? 'running' : 'paused'; c.style.opacity = waterOn ? '1' : '0.35'; })`
   - Работает для всех 16 элементов `.coilflow` (8 в rect + 8 в dist, но видимы только в активной группе)
   - Когда `waterValveOpen === false`: анимация на паузе, прозрачность 0.35
   - Когда `waterValveOpen === true`: анимация работает, полная прозрачность

8. **Бутыль в dist-режиме** — слева под дефлегматором, через `transform="translate(-232, 64)"` от исходной позиции
   - 3 капли продукта (`.product-layer-dist`) — видны когда `heaterOn` (конденсат капает при нагреве)

9. **Уменьшены отступы панели**: `padding: 14px 14px 8px` → `padding: 6px 6px 4px` для большего пространства под SVG

10. **CSS-анимация для pipe-vapor** (через offset-path) добавлена, но фактически используется `<animateMotion>` как более надёжный кросс-браузерный вариант

### Почему так решили
- **Дублирование бака в обеих группах** вместо `<use>` — потому что `<use>` с градиентами и class-селекторами для анимации работает нестабильно в некоторых браузерах. Дублирование надёжнее.
- **`<animateMotion>` для пара в П-трубе** вместо CSS `offset-path` — `animateMotion` поддерживается всеми браузерами с SVG, включая старые. CSS `offset-path` может не работать в некоторых рендерах.
- **localStorage для lastActiveProcess** — простое решение, переживает перезагрузку страницы. Если localStorage недоступен (приватный режим) — fallback на ректификацию.
- **`displayProcess` логика** — если процесс активен (1 или 2), показываем его и запоминаем. Если IDLE (0) — показываем последний запомненный. Это даёт пользователю возможность видеть, чем он занимался последний раз.
- **Капли продукта в дистилляции зависят от heaterOn** — в дистилляции нет клапана отбора, конденсат капает самотёком пока идёт нагрев. Когда нагрев выключен — капли скрыты.

#### Этап 3: Модификация ESP32 кода

**`SDLogger.h`** — новый метод `getLastLogLinesJson(int maxLines = 10)`:
- Читает только **1KB** с SD (вместо 16KB у `readLastLog`) — не нагружает систему
- Разбивает на строки, берёт последние 10 (новые сверху)
- Экранирует кавычки и обратные слэши
- Возвращает готовый JSON-массив: `["строка1","строка2",...]`
- Мьютекс с таймаутом 500мс — если SD занят, возвращает `[]`
- Старый метод `readLastLog()` сохранён без изменений

**`AppNetwork.cpp`** — добавлена одна строка перед `server->send()`:
```cpp
json += ",\"logs\":" + logger.getLastLogLinesJson(10);
```
Минимальное изменение, не влияет на остальной JSON.

### Почему так решили
- **1KB вместо 16KB** — `getLastLogLinesJson()` вызывается каждую секунду. Чтение 1KB с SD занимает ~1-2мс, чтение 16KB — ~20-30мс. Разница в 16 раз.
- **JSON-массив в `/api/status`** — логи идут вместе с остальными данными, не нужен отдельный fetch-запрос. Один запрос — одно соединение — меньше нагрузка на ESP32.
- **Скрытые span** — существующий JS (`setTemp('val-tsa-hud', ...)`) продолжает работать, температуры обновляются и на SVG, и в скрытых элементах. Не нужно менять логику `updateUI()`.
- **`bottleClipDist` внутри группы** — clipPath с `userSpaceOnUse` (по умолчанию) работает в локальной системе координат элемента. При размещении в `<defs>` координаты не совпадают после transform. Размещение внутри группы решает проблему.
